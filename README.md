## 📁 Each Problem Folder Contains

Every problem directory contains two files:

### 📄 README.md

The problem documentation includes:

- 📝 Problem Title
- 🔢 LeetCode Problem Number
- 📊 Difficulty (Easy / Medium / Hard)
- 🏷️ Tags
- 📖 Problem Description
- 📥 Input Format
- 📤 Output Format
- ✅ Example Test Cases
- ⚠️ Constraints
- 💡 Approach
- 🧠 Algorithm
- ⏱️ Time Complexity
- 💾 Space Complexity
- 🔍 Dry Run (if applicable)

### 💻 Solution.java

Contains:

- Clean Java solution
- Well-formatted code
- Meaningful variable names
- Inline comments (where necessary)
- Optimized implementation

# 🧠 Problem Solving Strategy

For every problem, I try to follow this order:

1. Understand the problem.
2. Identify constraints.
3. Think about brute-force.
4. Improve to optimal.
5. Analyze Time Complexity.
6. Analyze Space Complexity.
7. Write clean code.
8. Add comments if needed.
9. Push to GitHub.
